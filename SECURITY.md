# Security Policy

## Reporting a Vulnerability

When this repository is hosted on GitHub, report vulnerabilities that may
expose prompts, hook payloads, session identifiers, paths, credentials, or
local files through a private security advisory. Do not include sensitive
reproduction data in a public issue.

Non-sensitive hardening suggestions may use a regular issue. Include the
affected project, operating system, and version.

## Local Data

Each project must document what it reads, what it stores, retention behavior,
and how to remove local data. Collection of message content, raw hook payloads,
and credentials must be disabled by default.
